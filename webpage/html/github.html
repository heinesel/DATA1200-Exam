    Samarbeid i Github

    Bilder av commithistorier og endringer i skrift-estetikk.

    Git og GitHub

    Gruppen vår benyttet Git og Github som verktøy under prosjektet vårt. Git er en Open-source distribuert versjonskontrollsystem som muliggjør kontinuelig gruppesamarbeid på tvers av rom på en måte som unngår at vært medlemms bidrag ikke forstyrrer andre medlemmers bidrag (Robert, 2023). Derav unngår man å merge sin egen kode med andre sin på en uhensiktsmessig måte.

    Github er en skybasert samarbeidsplattform hvor utviklere kan lage, lagre, operere og dele kode. (GitHub Docs, 2024) Den plattformen er muliggjort igjennom verktøyet Git som er koblet til Github.

    Samarbeid

    Hver av gruppemedlemmene commitet til hovedgrenen flere ganger, og hver endring ble lagt til i commithistorien hvor vi hadde kontroll over hvilke endringer som hadde blitt gjennomført, og av hvem. Det er lettere å se endringer ved å sammenligne ulikhetene mellom forrige og nyere versjoner av koden. Eksempler på dette kan sees på bildene om commithistorien lenger opp på siden.

    Ved å se enda nøyere på dette kan man se etter spesifikke endringer. Eksempler på disse endringene kan være endringer i fontstørrelse, element-styling og ved å legge til html-kommentarer for klarere kode. Dette bidrar til å unngå kodefeil forårsaket av misforståelser.

    Fordeler

    Det er flere fordeler med å benytte Github i VSCode:

    • Integrert git:

      Gjør at man kan stage, commit, push, og pull inni samme IDE.

    • Live share:

      Gjør at flere utviklere kan programmere samtidig, og se endringer i sanntid.

    • Direktetilgang til Github repositories:

      Gjør at man enkelt kan klone og bruke andre repositories uten å forlate VScode.
